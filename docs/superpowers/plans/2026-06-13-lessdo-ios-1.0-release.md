# LessDo iOS 1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing LessDo Flutter prototype into a reliable, localized, free iPhone/iPad application that satisfies the approved iOS 1.0 release specification and App Store preparation gates.

**Architecture:** Keep Flutter and replace the current JSON-backed monolithic store with explicit domain models, SQLite repositories, an application controller, a notification coordinator, and a persistent focus controller. Platform plugins sit behind injectable interfaces so scheduling, biometrics, deep links, diagnostics, and UI behavior can be tested without invoking live system services.

**Tech Stack:** Flutter 3.41, Dart 3.11, Material 3, `sqflite`, `shared_preferences`, `flutter_local_notifications`, `flutter_timezone`, `timezone`, `local_auth`, `app_links`, `share_plus`, `url_launcher`, Flutter localization generation, XCTest, Flutter unit/widget/integration tests.

);

CREATE TABLE subtasks (
  id TEXT PRIMARY KEY,
  task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  title TEXT NOT NULL CHECK(length(trim(title)) > 0),
  completed INTEGER NOT NULL DEFAULT 0,
  sort_order INTEGER NOT NULL
);

CREATE TABLE focus_history (
  id TEXT PRIMARY KEY,
  task_id TEXT,
  task_title_snapshot TEXT NOT NULL,
  mode TEXT NOT NULL,

);

CREATE TABLE active_focus (

  payload_json TEXT NOT NULL
);

CREATE INDEX idx_tasks_list_order ON tasks(list_id, completed, sort_order);
CREATE INDEX idx_tasks_due ON tasks(completed, due_at_utc);
CREATE INDEX idx_tasks_reminder ON tasks(completed, reminder_at_utc);
